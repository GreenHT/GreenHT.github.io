Git学习笔记

本文最后更新于：4 天前

Git是一个分布式版本控制工具，与集中式的SVN区别开

Git的大致流程如图

  • 先从远程仓库将代码克隆（Clone）到本地仓库
  • 要干活的时候就把代码提取（checkout）到工作区，弄完以后暂存到暂存区，最后统一保存本地仓库里
  • 可以将成果同步（Push）到远程仓库
  • 可以把别人的修改同步（Pull）到本地

¶仓库管理基本操作（本地）

  1. 在目标文件夹中初始化仓库 git init

  2. 查看当前状态 git status

  3. 将修改完成的文件添加到暂存区 git add 文件名 git add 文件名1 文件名2 文件名3…… git add .(添加当前目录)

  4. 提交到本地仓库（版本库） git commit -m "注释内容" 注释内容为每次提交操作的记录

  5. 版本回退

    • 查看版本，确定版本号 git log git log --pretty=oneline

      图中的黄色字符串（即commit id）为版本号

    • 回退到某个版本 git reset --hard 版本号 版本号至少输4位

      **注意：**查看历史记录只能查看当前状态之前的版本，即回退之后较新的版本号将会查询不到，想撤销回退操作只能通过查看历史操作

    • 查看历史操作 git reflog

¶远程仓库操作

基于https协议：

  1. 将bash命令行cd到工作目录下（此步可省略）
  2. 复制远程仓库的https地址
  3. 使用git clone [https地址]
  4. cd到新生成的目录中
  5. 在仓库中做对应操作（提交到暂存区，提交到本地仓库，提交到远程仓库git push，拉取远程仓库git pull）

第一次向远程仓库提交时会提示无权限，需要向git的配置文件中添加用户名和密码信息

文件位置：./.git/config

修改内容：将url后面的地址中加上用户名和密码使得地址格式为https://用户名:密码@github.com/……

小技巧：每天第一件事为git pull，每天最后一件事为git push

注：新版的git只需要登陆一次，后续无需再添加用户名信息

基于SSH协议：

  1. 需要先安装opensshssh-keygen -t rsa -C "注册邮箱"

  2. 在GitHub中上传公钥

    若第一步中未修改的话，进入C:\Users\admin\.ssh\目录中，用文本编辑器打开id_rsa.pub文件，将内容复制到GitHub中（标题随意）

  3. 正常使用git（同https）

¶冲突的产生与解决

**冲突的产生：**在没有执行git pull的情况下修改内容并执行了git push

**解决：**先执行git pull，在完成内容重新整理（git会自动合并冲突）后再git push

¶忽略文件

有一些文件万年也不会动的，并不需要每次提交；或者有一些文件我们并不想提交，可以使用忽略文件

方法： 在本地仓库的目录下新建文件名为.gitignore，内容为需要忽略的文件和文件夹

注意：Windows下可能无法创建此文件，可以使用bash命令行通过touch创建

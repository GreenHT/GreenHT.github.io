Hexo搭建Github Page实录

本文最后更新于：1 天前

今天研究了下用hexo框架搭建Github Page，按照网上的教程来弄或多或少有一些问题，也没有找到合适的完整的解决方法，所以自己记录了一下过程

¶安装环境

不管是在Windows环境下还是Linux环境下都要先安装Node.js和Git，安装过程略，这里记录一下Windows环境

¶安装Hexo

npm install -g hexo-cli

这个步骤打开命令行执行就行，安装文件会下载到Nodejs的默认目录，不会下载到命令行的执行目录的

安装速度慢或甚至连接不上的可以换用npm的淘宝源

npm config set registry https://registry.npm.taobao.org

这里要把hexo添加到环境变量中，参考https://blog.csdn.net/qq_35561857/article/details/81590953

¶初始化Hexo

安装完hexo后，将命令行cd到工作目录，所有博客的网页文件都会放在这个目录里，执行

hexo init

执行结束后再执行

npm install

¶查看网页效果

执行hexo g后再执行hexo s，如果第二个命令遇到端口占用的问题可以换端口hexo s -p 端口号

然后会提示打开http://localhost:端口号查看，按Ctrl+c会停止网站

¶Hexo基本配置

配置文件在工作目录下的_config.yml，大部分参数官网上都有https://hexo.io/zh-cn/docs/configuration

¶主题

将找到的主题放在工作目录的theme文件夹下，文件夹名字为主题名，在hexo配置文件中的theme后面改成主题名。主题的独立配置文件在主题文件夹下的_config.yml

¶常见问题及解决

这里列举一些我遇到的问题

  1. 部署到GitHub后能打开页面但加载不出来任何css和js，即页面无法加载（当然网络的问题除外）

    解决办法：在hexo配置文件中的URL处（靠近开头的位置），url后面是GitHub Page的地址，root后面是斜杠加仓库名（/xxx.github.io）

  2. hexo转换md为html时，文件后缀的html消失（可能是转换出问题），导致打开网页时浏览器识别为下载文件

    解决办法：hexo的配置文件中
